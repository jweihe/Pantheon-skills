# Pantheon Experiment Rubric

Use experiments to prove that a skill is practical, not merely elegant.

## Required Claims

A validation report should answer:

- **Usefulness**: What repeated work became easier?
- **Specificity**: What does this skill know that a generic agent would not?
- **Reliability**: What deterministic checks passed?
- **Boundaries**: What can the skill propose but not do without confirmation?
- **Transfer**: Would the skill still help on a second, similar task?

## Minimum Experiment

1. Start with a short messy brief or transcript.
2. Distill it into a skill proposal.
3. Scaffold the skill in a temporary directory.
4. Audit the generated skill.
5. Record failures and revise.

## Scoring

Score each category from 0 to 2:

- **Trigger clarity**: 0 vague, 1 usable, 2 precise with concrete contexts.
- **Workflow leverage**: 0 generic, 1 somewhat reusable, 2 clearly shortens future work.
- **Resource fit**: 0 missing or cluttered, 1 acceptable, 2 resources match repeated operations.
- **Validation integrity**: 0 untested, 1 superficial checks, 2 checks can catch real defects.
- **Bounded autonomy**: 0 unsafe mutation, 1 partially bounded, 2 reviewable and consent-based.

A skill needs at least 8 of 10 before installation is recommended.
